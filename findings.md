# 项目调研发现

## 后端 API 完成度

经分析 11 个后端模块，**后端 API 整体完成度约 90%**：

### 已完成（可直接使用）
- 用户认证：注册/登录/刷新Token/获取用户信息
- 题库管理：CRUD + 搜索/筛选/批量删除
- 标签管理：树形标签 + 29个预设标签
- 试卷管理：手动组卷 + 智能组卷 + 列表/详情/删除
- OCR 识别：图片上传 → Pix2Text 识别 → 结果保存/修正/历史
- Word 导出：试题卷 + 答案卷（化学式Unicode转换）
- 本地存储降级：未配置COS时自动使用本地文件

### 部分完成
- 微信登录：API 代码完整但 openid 用占位符
- OCR 预处理：函数已写但未集成到识别流程
- Word OMML：简单化学式转Unicode OK，复杂公式用占位符
- 试题列表：不返回标签（懒加载问题已简化处理）

### 未完成
- 试卷更新接口
- 标签更新接口
- 学生端全部API（MistakeBook/PracticeRecord 有模型无接口）
- 前端应用（零代码）

## 技术架构

```
本地开发环境:
  SQLite (自动建表) → FastAPI (async) → uvicorn
  COS降级为本地文件存储

生产环境 (规划):
  MySQL 8.0 → FastAPI → Nginx → 腾讯云
  Redis (JWT黑名单/限流)
  Pix2Text OCR 微服务 (Docker)
  腾讯云 COS (图片/文件存储)
```

## 关键技术点

### 化学公式处理链路
```
存储: LaTeX格式 ($H_2SO_4$ + $NaOH$)
  ↓
Word输出: Unicode下标 (H₂SO₄ + NaOH)
  ↓  
小程序显示: 需要 KaTeX/mp-html 渲染
```

### 安全措施
- JWT密钥随机48位
- 速率限制：登录10次/分钟，全局可配
- CORS白名单
- 安全响应头
- Swagger可关闭
- 监听127.0.0.1仅本地访问
