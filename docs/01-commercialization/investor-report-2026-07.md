## 一、开场：我们在解决什么问题

各位投资人好。

今天汇报的项目叫"小睿化学"，全称是"高中化学教学辅助系统"。我们面向的群体很具体——高中化学老师。

化学老师的日常工作里，有一大块重复劳动：找题、录题、整理题库、组卷、排版。一道带化学公式、装置图、实验流程的题目，从拍照或截图到变成一份能直接打印的 Word 试卷，中间要经过识别、录入、排版、校对多个环节。很多老师至今还在用微信收藏、Excel、甚至手动复制粘贴的方式管理题目。

我们的目标很简单：让老师用手机拍一道题，系统在几十秒内完成识别、入库、组卷，并导出一份排版整齐的 Word 试卷。

---

## 二、产品定位与核心价值

我们给自己的定位不是"又一个题库App"，而是**老师个人题库的生产力工具**。

核心价值主张：

1. **拍题即入库**：用手机拍照，OCR 自动识别题干、选项、公式，老师只做复核。
2. **个人题库沉淀**：题目按教材、知识点、难度、题型分类，长期积累成数字资产。
3. **一键组卷导出**：手动或自动选题，直接生成 Word，试题卷和答案卷分离，支持附图。
4. **微信生态闭环**：小程序内完成识别、编辑、组卷，Word 文件可直接转发到文件传输助手，老师在电脑端继续编辑打印。

整个使用链路控制在"拍照 → 复核 → 组卷 → 导出"四步以内，单题入库时间从传统的十几分钟降到一分钟以内。

---

## 三、目标市场

**第一阶段目标用户**：中国大陆高中化学老师，以及部分初中化学老师和教培从业者。

选择这个切口的原因：

- **需求真实且高频**：组卷、出题是老师每周甚至每天都在做的事。
- **付费意愿明确**：老师愿意为"省时间"的工具付费，尤其是能减少机械劳动的产品。
- **竞争相对分散**：现有产品要么是面向学生的搜题软件（作业帮、小猿搜题），要么是面向学校采购的重型题库系统。个人老师好用的轻量工具不多。
- **内容可沉淀**：老师一旦开始录题，题库就是他的数字资产，迁移成本高，用户粘性自然形成。

## 四、解决方案：产品与技术架构

### 4.1 前端：微信小程序

- 基于 uni-app + Vue 3 开发，主力运行在微信小程序。
- 覆盖教师端核心链路：登录、OCR、编辑入库、题库、组卷、导出、分享。
- 学生端配套：错题本、练习记录。

### 4.2 后端：FastAPI + SQLAlchemy 2.0

- Python FastAPI 异步框架，SQLAlchemy 2.0 async session。
- 数据库：开发用 SQLite，生产用 MySQL 8.0，已通过 Alembic 引入迁移治理。
- 部署：腾讯云 CloudRun 容器托管，本地 Docker Compose 可一键启动。
- 认证：JWT + 微信登录 OpenID。

### 4.3 OCR 多引擎策略

我们没有依赖单一 OCR，而是做了三引擎分层调度：

| 引擎 | 定位 | 适用场景 |
|---|---|---|
| tesseract | 极速识别 | 纯文本、简单题、零成本 |
| pix2text_online | 高精度公式 | 含化学式、方程式的题目 |
| doubao_vision | 复杂题识别 | 流程图、装置图、复杂混排 |

系统按题目类型和成本自动推荐引擎，并对高成本引擎做用户级和全局每日额度控制，避免成本失控。

### 4.4 Word 导出

- 后端用 python-docx 生成 Word。
- 化学式统一以 LaTeX 存储，导出时转换为 Unicode 上下标。
- 支持题目附图嵌入（实验装置图、分子结构图等），试卷卷和答案卷分离。
- 小程序端通过微信 shareFileMessage 直接转发 Word 文件。

### 4.5 个人开发者控制台

除了老师用的小程序，我们还落地了一个桌面端管理后台，供日常维护：

- 系统概览、题库管理、标签管理、OCR 记录、试卷管理、用户管理、成本监控、Billing 运营。
- 通过 JWT + 白名单控制访问。
- 后台已接入种子计划的资格、订单、权益管理。

## 六、商业模式

我们的商业化路径分两步走：

### 第一阶段：种子计划（底座已落地）

- 前 10 名用户：终身免费。
- 第 11-50 名用户：9.9 元终身权益。
- 目标：快速获取种子老师，验证付费意愿，收集真实使用反馈。

### 第二阶段：套餐订阅

- 基于用户 OCR 用量、题库容量、导出次数设计分层套餐。
- 高成本 OCR 引擎按额度售卖，普通文本识别保持低价或免费额度。
- 后台已支持为用户配置每日/月度 OCR 限额和套餐来源，为后续正式套餐体系打好底座。

### 成本结构

- 主要成本来自 OCR API 调用和云托管资源。
- 已落地用户级和全局额度控制，防止单个用户或突发流量击穿成本。
- 后续会基于 1-2 周真实运行数据，决定继续 CloudRun 还是迁移到 CVM 固定成本。

## 七、竞争优势

1. **切口精准**：只做高中化学老师，不做泛教育、不做学生端主战场。
2. **多引擎 OCR**：化学式、装置图、流程图都能识别，不是简单文字 OCR。
3. **微信原生闭环**：识别、编辑、组卷、导出、转发全部在微信生态内完成。
4. **个人题库资产化**：题目属于老师个人，长期使用形成粘性。
5. **工程化意识**：从早期就重视配置治理、成本控制、数据库迁移、测试覆盖，不是一次性 demo。
