# 高中化学教学辅助系统开发计划

## 1. 项目目标

面向高中化学老师，做一套以“小程序快速录题 + 题库管理 + 试卷导出”为核心的教学辅助工具。

当前产品主线已经明确为：

1. 老师拍题或上传图片
2. 选择合适的识别引擎完成 OCR
3. 在识别结果页做快速复核和小改
4. 需要完整维护时进入编辑入库页
5. 将题目沉淀到题库，后续用于筛题、组卷、导出 Word

## 2. 当前技术路线

### 后端

- FastAPI
- SQLAlchemy 2.0
- SQLite（开发）/ MySQL（生产）
- 腾讯云 COS 存储

### 前端

- uni-app
- 微信小程序

### OCR

- `tesseract`
  - 定位：极速识别
  - 场景：纯文本、轻量、快速录入
- `pix2text_online`
  - 定位：高精度识别（公式）
  - 场景：公式较多、版式相对清晰的题目
- `doubao_vision`
  - 定位：高精度识别（复杂题）
  - 场景：复杂混排、流程图、装置图、整页理解

## 3. 产品职责划分

### 识别结果页

保留，职责收口为：

- 查看原图
- 查看识别结果
- 修改少量错字、漏字
- 无明显问题时快速保存
- 有复杂问题时进入编辑入库

不再承担的职责：

- 大段技术提示
- 重复的结构化说明
- 过重的完整编辑流程

### 编辑入库页

承担完整维护职责：

- 完整编辑题目正文
- 维护答案
- 维护解析
- 维护题型、难度、标签
- 维护题目附图
- 提供“补拍附图”入口

## 4. 题目附图策略

### 设计原则

对于流程图、装置图、结构图这类内容：

- 不强行把图全部转成文字
- 正文识别负责题干与选项
- 图尽量作为附图保留

### 两条附图链路

#### 链路 A：OCR 已有附图

- OCR 结果中如果已经带出题目附图
- 进入编辑入库页后，附图应继续保留
- 保存题目时，附图要真正挂接到题目

#### 链路 B：老师手动补拍

- 如果 OCR 无法可靠提取附图
- 在编辑入库页提示老师补拍附图
- 老师可直接调起相机单独拍题目附图
- 保存后题目详情页可查看附图

## 5. 当前已经确定的交互规则

### 引擎文案

以业务文案为主，技术名做小字说明：

- 极速识别 / Tesseract
- 高精度识别（公式） / Pix2Text
- 高精度识别（复杂题） / 豆包视觉

### 识别质量

只做弱提示，不做强指标：

- 识别质量：较高
- 识别质量：一般
- 识别质量：需复核

并附带：

- 仅供参考

### 补拍附图提示

由规则判断触发，而不是直接展示模型原始说明：

- 模型判断题目存在图，且建议保留原图
- 或正文中出现“如图 / 流程图 / 装置图 / 结构图”等关键词
- 且当前题目还没有附图

命中时前端展示：

- 建议补拍附图
- 去补拍附图

## 7. 里程碑顺序

### Phase A：结果页收口

- 统一结果页和编辑页正文规则
- 弱化置信度
- 去掉无价值的结构化提示
- 做清晰的“建议人工复核 / 建议补拍附图”

### Phase B：附图链路稳定

- OCR 附图稳定保留
- 编辑页补拍附图
- 详情页查看附图
- 保存链路稳定

### Phase C：云端真机验证

- 新镜像部署
- 验证 OCR 结果页
- 验证编辑入库保存
- 验证题目详情附图展示

### Phase D：附图导出

- Word 导出时插入题目附图
- 处理图文顺序和版式
- 验证试卷与答案导出效果

## 8. 当前最优先事项

1. 部署本轮新版本到云端
2. 做真机测试，重点验证：
   - 结果页正文是否完整
   - 编辑页正文是否一致
   - 补拍附图是否可保存
   - 详情页附图是否可查看
3. 开始设计 Word 导出附图方案
