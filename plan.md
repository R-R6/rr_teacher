### 2.3 OCR 方案详解（关键！）

化学题目的 OCR 是整个项目最难的部分。推荐**双引擎策略**：

```
用户拍照上传
      │
      ▼
┌──────────────┐
│ 图片预处理     │  → 裁剪、增强对比度、去噪
└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌─────────────────┐
│ Pix2Text (P2T)│────▶│ 混合识别结果      │
│ 中文 + 化学式  │     │ 含 LaTeX + 中文   │
│ 混合识别      │     └────────┬────────┘
└──────────────┘              │
                              ▼
┌──────────────┐     ┌─────────────────┐
│ LaTeX-OCR    │────▶│ 纯公式区域       │
│ 纯公式识别    │     │ 增强识别         │
└──────────────┘     └────────┬────────┘
                              │
                              ▼
                     ┌─────────────────┐
                     │ 结果合并 + 结构化  │
                     │ LaTeX 统一格式    │
                     └────────┬────────┘
                              │
                              ▼
                     ┌─────────────────┐
                     │ 前端渲染预览      │
                     │ (mp-html 渲染)   │
                     │ 用户可手动修正    │
                     └─────────────────┘
```

**Pix2Text** 是当前最好的中文 + 公式混合 OCR 开源方案，支持识别：
- `H₂SO₄ + NaOH → Na₂SO₄ + H₂O`（化学方程式）
- `Fe²⁺` `SO₄²⁻`（离子符号/上下标）
- 苯环结构式（简单有机结构）
- 反应条件（`Δ` `催化剂` 等）

**局限性**：
- 复杂实验装置图的结构化识别需要额外处理（可能保留为图片）
- 化学结构式（键线式）识别准确率有限
- 建议：结构式/装置图区域 → 保留原图嵌入 Word，文字和公式区域 → OCR 转电子版

## 四、关键技术难点 & 解决方案

### 难点 1：化学公式 OCR 准确率

**方案**：Pix2Text 为主引擎 + LaTeX-OCR 为辅助引擎（处理纯公式区域），OCR 结果提供「一键复制原图区域」按钮——结构式/装置图直接保留图片嵌入 Word。

### 难点 2：LaTeX → Word 化学公式对象

**方案**：使用 `latex2mathml` 将 LaTeX 转为 MathML，再通过 python-docx 的 OMML（Office Math Markup Language）写入 Word。Word 原生支持 OMML，化学式在 Word 中可直接编辑。

### 难点 3：微信小程序 LaTeX 渲染

**方案**：使用 `mp-html` 组件 + KaTeX 渲染 LaTeX 公式，或使用 `katex-mini` 微信小程序专用版本。

### 难点 4：中老年教师操作体验

**方案**：
- 大按钮 + 大字体（最小 16px）
- 三步完成核心流程：拍照 → 查看 → 导出
- 首次使用引导动画
- 关键操作二次确认（防止误删）

## 八、OCR 产品路线更新（2026-06-18）

基于当前真实测试结果，OCR 路线更新为：

### 8.1 模式分工

| 模式 | 目标 | 引擎 | 适用场景 |
|------|------|------|----------|
| 极速识别 | 低成本快速草稿 | `tesseract` / `pix2text_online` | 纯文本题、无附图题、快速录入 |
| 高精度识别 | 高复杂度题结构化录入 | `doubao_vision` | 流程图、装置图、混排复杂题 |

### 8.2 附图处理原则

对于复杂题中的图片：

- 不要求豆包把图片强制识别成文本
- 图片应优先作为“题目附图”独立保留
- 理想路径：
  1. 自动裁切题目附图区域
  2. 裁切失败时，允许老师在编辑入库页中单独补拍附图
  3. 题目详情页显示附图
  4. Word 导出时带附图

### 8.3 结果页职责调整

结果页的核心功能应为：
- 展示原题图片
- 展示可编辑的识别结果
- 让老师快速修正后保存

因此当前“LaTeX识别结果”区域，产品语义上应逐步调整为：
- `识别结果（可编辑）`

只有在 `pix2text_online` 等偏公式引擎场景下，才保留“原始公式结果”的概念。

### 8.4 后续开发顺序

推荐按以下里程碑推进：

1. **结果页收口**
   - 统一可编辑正文来源
   - 减少重复预览
   - 弱化 LaTeX 命名

2. **豆包高精度真机可用**
   - 云端部署
   - 引擎切换稳定
   - 结构化结果可回填

3. **题目附图自动裁切**
   - 候选区域提取
   - 保存到 `QuestionImage`
   - OCR 记录与题目关联

4. **编辑入库页补拍附图**
   - 单独调起相机
   - 上传并挂接到题目

5. **题目详情 + Word 导出带图**
   - 详情页点击放大
   - 导出试卷带附图
