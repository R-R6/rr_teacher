## 本轮完成（2026-06-22）

### 1. 识别结果页

- 结果页不再用“选最长字符串”这种不稳定逻辑，而是新增统一正文规则：
  - 优先使用 `structured.question_text`
  - 自动拼接结构化选项
  - 必要时回退到更完整的 `blocks / result_text / result_latex`
- 结果页标题调整为更贴近老师使用场景的文案：
  - `题目正文（可快速修改）`
  - `公式识别结果（参考）`
- 识别引擎文案调整为业务名称优先：
  - `极速识别 / Tesseract`
  - `高精度识别（公式） / Pix2Text`
  - `高精度识别（复杂题） / 豆包视觉`
- 识别质量改为弱提示：
  - `识别质量：较高 / 一般 / 需复核`
  - 附带 `仅供参考`
- 新增“建议人工复核 / 建议补拍附图”提示卡
- 原图查看入口保留并明确显示为 `查看原图`

### 2. 编辑入库页

- 编辑页接入与结果页相同的正文规则，避免两页正文不一致
- 保留完整编辑能力：
  - 题目内容
  - 答案
  - 解析
  - 题型
  - 难度
  - 标签
- 新增题目附图维护区：
  - 查看当前附图
  - 删除附图
  - `补拍附图`
  - `从相册添加`

### 3. 题目附图保存链路

- 修复了从 OCR 进入编辑页时，OCR 附图可能没有真正挂到题目上的问题
- 现在创建/编辑题目时，OCR 附图和手动补拍附图都会一起提交
- 后端会把 OCR 既有附图正确关联到当前题目，并保留排序与类型

### 4. 新增可验证规则模块

- 新增前端工具文件：
  - `frontend/src/utils/ocr-result-helpers.mjs`
- 已覆盖的规则测试：
  - 正文优先使用结构化题干并拼接选项
  - 无结构化题干时回退到更完整内容
  - 低置信度/人工复核时给出“需复核”
  - 有图但未挂图时给出“建议补拍附图”
